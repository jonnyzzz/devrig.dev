# Features

devrig is a command-line tool that helps you set up and manage development environments.

## Execution Modes

devrig supports two execution modes:

### Per-Project Mode (Recommended)

Each project has its own configuration and isolated environment:

- **Project Configuration**: `devrig.yaml` file in your project root
- **Isolated Dependencies**: Each project can use different IDE and tool versions
- **Version Control**: Configuration is part of your repository
- **Bootstrap Scripts**: Cross-platform scripts (`devrig`, `devrig.bat`, `devrig.ps1`) for easy execution

**Getting started:**

```bash
# Download devrig binary for your platform
# See the Download page for links

# Initialize devrig in your project
./devrig init

# Commit the configuration
git add devrig devrig.bat devrig.ps1 devrig.yaml
git commit -m "Add devrig configuration"
```

### Machine-Wide Mode

Install devrig globally to use it across all projects:

- Install devrig binary to your PATH
- Run `devrig` commands from anywhere
- Useful for installing system-wide tools like fonts

## Available Commands

### `devrig init`

Initialize devrig in a project directory:

```bash
devrig init [directory]
```

**What it creates:**
- Bootstrap scripts: `devrig`, `devrig.bat`, `devrig.ps1`
- Configuration file: `devrig.yaml`
- Local binary in `.devrig/` directory

**Options:**
- `--scripts-only` - Generate only bootstrap scripts
- `--init-from-local` - Use current binary instead of downloading

### `devrig install jetbrains-mono`

Install JetBrains Mono font:

```bash
devrig install jetbrains-mono
```

**What it does:**
- Downloads latest version from GitHub
- Installs to system font directory:
  - Windows: `%WINDIR%\Fonts`
  - macOS: `~/Library/Fonts`
  - Linux: `~/.local/share/fonts/JetBrainsMono`
- Refreshes font cache on Linux (`fc-cache -f`)

### `devrig version`

Show the current version:

```bash
devrig version
```

## Platform Support

devrig runs on:

- **Windows** (x86_64, ARM64)
- **Linux** (x86_64, ARM64)
- **macOS** (ARM64 / Apple Silicon)

All commands work in:
- Bash
- PowerShell
- cmd.exe

## Configuration File

The `devrig.yaml` file stores project configuration:

```yaml
devrig:
  version: 0.79.0
  release_date: "2025-11-20"
  binaries:
    darwin-arm64:
      url: "https://github.com/jonnyzzz/devrig.dev/releases/..."
      sha512: "ce6bba63..."
    linux-x86_64:
      url: "https://github.com/jonnyzzz/devrig.dev/releases/..."
      sha512: "cdaa595e..."
    # ... other platforms
```

This file is generated by `devrig init` and can be committed to version control.
